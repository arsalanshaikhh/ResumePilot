## Task 2: Fix openRouterService — three crash paths in parseResponse

**Files:**
- Modify: `src/services/openRouterService.ts:119-141`

Three bugs in `parseResponse`:
1. `choices[0]` throws if the array is empty (content-filtered responses)
2. `JSON.parse(content)` throws if the LLM wraps its JSON in markdown fences (e.g. ` ```json\n...\n``` `)
3. `usageData.prompt_tokens` throws if the API omits the `usage` field

- [ ] **Step 1: Replace the entire parseResponse method**

In `src/services/openRouterService.ts`, replace lines 119–141:

```ts
private parseResponse(data: unknown, _analysisType: 'ats' | 'match'): AnalysisResponse {
  try {
    const responseData = data as {
      choices?: Array<{ message: { content: string } }>
      usage?: { prompt_tokens: number; completion_tokens: number; total_tokens: number }
    }

    if (!responseData.choices || responseData.choices.length === 0) {
      return { success: false, error: 'API returned no response choices' }
    }

    let content = responseData.choices[0].message.content

    // Strip markdown code fences that LLMs sometimes wrap JSON in
    const fenceMatch = content.match(/```(?:json)?\s*([\s\S]*?)```/)
    if (fenceMatch) {
      content = fenceMatch[1].trim()
    }

    const parsed = JSON.parse(content)

    const usageData = responseData.usage
    return {
      success: true,
      data: parsed,
      usage: usageData
        ? {
            promptTokens: usageData.prompt_tokens,
            completionTokens: usageData.completion_tokens,
            totalTokens: usageData.total_tokens,
            cost: 0,
          }
        : undefined,
    }
  } catch {
    return {
      success: false,
      error: 'Failed to parse API response',
    }
  }
}
```

## Task 4: Fix ScoreChart inverted react/newPlot condition

**Files:**
- Modify: `src/components/ats/ScoreChart.tsx:340`

The condition `hasData && !dataChanged` calls `Plotly.react()` (incremental update) when data has **not** changed, and `Plotly.newPlot()` (expensive full re-render) when data **has** changed. The intent is the opposite: use `react()` for updates, `newPlot()` only for the initial render.

- [ ] **Step 1: Fix the condition**

In `src/components/ats/ScoreChart.tsx`, replace lines 340–344:

```ts
// Before
if (chartElement && hasData && !dataChanged) {
  window.Plotly.react(chartElement, plotData, layout, config)
} else if (chartElement) {
  window.Plotly.newPlot(chartElement, plotData, layout, config)
}

// After
if (chartElement && hasData && dataChanged) {
  window.Plotly.react(chartElement, plotData, layout, config)
} else if (chartElement) {
  window.Plotly.newPlot(chartElement, plotData, layout, config)
}
```
